Custom Cursor System Guide

Overview

The By Default website features a sophisticated custom cursor system that replaces the default browser cursor with a branded, interactive cursor experience. The system is built with CSS and JavaScript, using GSAP for smooth animations.

Features

- Desktop-only: Only activates on screens 991px and wider

- Progressive enhancement: Gracefully falls back to default cursor if JavaScript/GSAP fails

- Smooth animations: Uses GSAP for fluid cursor following

- Multiple cursor states: 20+ different cursor types for various interactions

- Nested element support: Child elements can override parent cursor states

- Click feedback: Visual feedback when clicking

- Branded design: Uses the brand green color and custom SVG icons

- Accessibility-first: Never leaves users without a visible cursor

JavaScript API

Automatic Detection

The cursor automatically detects the appropriate state based on the element under the mouse pointer:

javascript



              


                // Automatically resolves cursor type at mouse position


function getCursorTypeAtPoint(x, y) {


  const el = document.elementFromPoint(x, y);


  if (!el) return null;


  const host = el.closest("[data-cursor]");


  return host ? host.getAttribute("data-cursor") : null;


}

Manual Control

You can manually set cursor states:

javascript



              


                // Set cursor type programmatically


function setCursorType(type) {


  const cursor = document.querySelector('.cursor-default');


  if (type) {


    cursor.classList.add("cursor-custom", cursor-${type});


  } else {


    cursor.classList.remove("cursor-custom");


  }


}

Animation Details

GSAP Configuration

The cursor uses GSAP for smooth following animations:

javascript



              


                // Main cursor follows with slight delay


gsap.to(cursor, {


  x: mouseX,


  y: mouseY,


  duration: 0.25,


  ease: "power3.out",


  overwrite: "auto",


});



// Halo follows immediately


gsap.to(cursorHalo, {


  x: mouseX,


  y: mouseY,


  duration: 0.1,


  ease: "none",


  overwrite: "auto",


});

Progressive Enhancement

The custom cursor system uses a progressive enhancement approach to ensure accessibility:

1. Default state: Normal browser cursors are shown by default

2. JavaScript check: Only when JS loads successfully does it hide default cursors

3. GSAP dependency: Verifies GSAP is available before initializing

4. Graceful fallback: If any dependency fails, users keep their default cursor

This ensures users are never left without a visible cursor, regardless of JavaScript availability or library loading issues.

Browser Support

- Desktop only: Only activates on screens ≥991px wide

- Modern browsers: Requires CSS custom properties support

- GSAP dependency: Requires GSAP library for animations

- Progressive enhancement: Works without JavaScript (shows default cursors)

Performance Considerations

- Hardware acceleration: Uses will-change: transform for optimal performance

- Event throttling: Mouse move events are handled efficiently

- Memory management: Cursor state is cached to prevent unnecessary updates

Troubleshooting

Cursor Not Appearing

- Ensure .cursor-default and .cursor-halo elements exist in HTML

- Check that screen width is ≥991px

- Verify GSAP library is loaded

- Check browser console for initialization warnings

- If custom cursor fails, default browser cursor should still be visible

Cursor States Not Working

- Check data-cursor attribute spelling

- Ensure cursor type exists in CSS

- Verify element is not being overridden by child elements

Performance Issues

- Check for excessive mouse move event handlers

- Ensure GSAP is using hardware acceleration

- Monitor for memory leaks in cursor state management
