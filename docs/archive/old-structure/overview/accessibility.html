Accessibility

Learn how to make the most of ByDefault's accessibility features and guidelines.

Overview

ByDefault is built with accessibility as a core principle. All components follow WCAG 2.1 AA guidelines and are designed to work seamlessly with assistive technologies. Our neurodiverse-led approach ensures that interfaces are inclusive by default.

✨ Key Features

  • • Full keyboard navigation support
  • • Screen reader compatibility
  • • High contrast mode support
  • • Reduced motion preferences respected
  • • Focus management and trapping
  • • ARIA attributes and semantic HTML

Keyboard Navigation

All interactive components support full keyboard navigation. Users can navigate through interfaces using only the keyboard, making the components accessible to users with motor disabilities and those who prefer keyboard shortcuts.

Common Keyboard Patterns

Arrow Keys

Navigate through menu items, tabs, and list options

↑ ↓ ← →

Enter/Space

Activate buttons and select options

Enter or Space

Escape

Close modals and popovers

Escape

Tab

Move focus between interactive elements

Tab / Shift+Tab

Screen Reader Support

All components include proper ARIA attributes and semantic HTML to ensure screen readers can accurately describe the interface to users.

ARIA Implementation

Labels and Descriptions

<button aria-label="Close dialog">
  <CloseIcon />
</button>

Live Regions

<div aria-live="polite" aria-atomic="true">
  Status updates appear here
</div>

State Management

<button aria-expanded="true" aria-controls="menu">
  Open Menu
</button>

Focus Management

Proper focus management ensures users can navigate interfaces logically and efficiently. ByDefault components handle focus trapping, restoration, and visual indicators.

Focus Indicators

All interactive elements include visible focus indicators that meet WCAG contrast requirements. 

/* Focus ring with high contrast */
.focus-visible {
  outline: 2px solid #2563eb;
  outline-offset: 2px;
}

Motion and Animation

ByDefault respects user preferences for reduced motion and provides alternative experiences for users with vestibular disorders or motion sensitivity.

Reduced Motion Support

@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}

When users have reduced motion preferences enabled, animations are minimized or removed entirely.

Color and Contrast

All color combinations meet WCAG 2.1 AA contrast requirements (4.5:1 for normal text, 3:1 for large text). Information is never conveyed through color alone.

High Contrast Mode

Components adapt to system high contrast settings

@media (prefers-contrast: high) {
  .component {
    border: 2px solid;
  }
}

Color Independence

Information is conveyed through multiple visual cues

<span class="status error">
  <Icon name="error" />
  Error message
</span>
